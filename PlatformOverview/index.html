Interfaces
Everything we see and hear is coming from some king of interface.
  • Interface examples: facebook feed, subreddits, twitter feed
Current interfaces problems
Interfaces gather our attention and direct it according to their owners decisions to maximize profits according to its monetization model.
  • Example: `You might also like` section on any webpage, `Recommended` etc all of those are there because it's beneficial for site owner to keep you longer on site or give you more buying options
  • Current monetization models are mostly ad driven
  • Attention is extracted mercilessly because the risks for extracting attention are pushed back in time
  • Current interfaces don't want to answer question: `Why do I see this?`
  • We want change that and make it possible to investigate exactly what data were used and what algorithms to display information that you see.
Link Exchange - Interface that tells you exactly why you are seeing given link.

Demo:
http://rinkeby.etherscan.io
  • LinkExchange is a solution for very specific problem and does not exhaust possibilities of Userfeeds Platform
Interface Examples:
  • Attention Guard
  • Governance
  • News
  • Attention Guard - interface showing messages from given sources, but only if a message is accompanied by economic sacrifice. (eg. holding tokens fro long time, simple payment, etc.)
  • Governance - Showing vote results among all token holders each with power equal to token holdings. (with additional restrictions, blacklisted addresses/chains of addresses)
  • News - Items sorted by accumulated score based on token holdings/transaction history/other data
  • Each of those interfaces will require customized algorithm, but after first algorithm is created all interfaces can use it
Data enrichment: Data Scientists will be able to download whole dataset and process it in ways that can be done only offline. Later they will be able to post results of their processing as `claims` back to dataset for `algorithms` use.
Example analysis: fraud detection and address blacklisting (eg. hacking related addresses)
